## 一、系统概述

### 1.1 平台定位

信速应是一个**短信中间商平台**，对下（甲方客户）提供 HTTP 接口收短信 + 客户端 Web 平台，对上（运营商/上游通道）用 CMPP/SGIP/SMGP 或 HTTP 连接发送，中间做路由、计费、客户管理。

```
甲方（企业客户）──HTTP──> 【本平台】──CMPP/HTTP──> 运营商 / 上游通道
   (给钱)                 (中间商)                   (实际发送)
```

### 1.2 三平台架构

| 平台 | URL | 端口 | 职责 |
|------|-----|------|------|
| 运营管理 | http://36.139.116.238:8889 | 8889 | 平台运营管理员使用 |
| 客户端 | http://36.139.116.238:8890 | 8890 | 企业客户使用 |
| 发送网关 | http://36.139.116.238:8888 | 8888 | 对外 HTTP 接口 |

### 1.3 默认账号

| 角色 | 账号 | 密码 | 权限范围 |

| 超级管理员 | admin | 123456 | 全部功能 |

#### 3.4.1 业务看板

**功能**：一站式数据总览

**包含内容**：
- **8 个 KPI 卡片**：合作方总数、有效合作期限、待处理投诉、拒收记录、通道总数、产品总数、发送成功率、今日发送量
- **4 个图表**：投诉趋势、处理效率仪表盘、合作方投诉排行、投诉来源分布
- **8 个快捷入口**：跳转各台账模块

**操作**：业务管理 → 业务看板 → 查看数据

#### 3.4.2 合作期限台账

**功能**：跟踪合作期限，预警到期

**特性**：
- 列表按"距到期天数"高亮（红/橙/绿）
- 状态代替删除（终止 = 状态置为 0）
- 资质过期的企业无法新增合作期限

**操作流程**：
1. 业务管理 → 合作期限台账
2. 新增：点击「新增」→ 选择企业/端口 → 填写开始/结束日期 → 保存
3. 终止：选择记录 → 点击「终止」→ 确认
4. 查看统计：业务管理 → 合作期限统计

#### 3.4.3 投诉处理

**功能**：投诉全流程管理

**状态流转**：待处理(0) → 处理中(1) → 已处理(2)

**投诉来源**：运营商 / 用户 / 监管

**操作流程**：
1. 业务管理 → 投诉处理
2. 登记投诉：点击「新增」→ 填写企业/号码/内容/来源/时间 → 保存
3. 处理投诉：选择记录 → 点击「处理」→ 更新状态/处理人/处理结果 → 保存
4. 查看详情：选择记录 → 点击「详情」
5. 查询：按企业/端口/号码/来源/状态筛选

#### 3.4.4 拒收记录

**功能**：查看用户拒收（TD/退订）记录

**数据来源**：上行短信含 T/TD/退订 关键词时，系统自动识别并记录

**字段**：企业编号、端口、子端口、拒收号码、拒收内容、拒收时间、Inbox ID

**操作流程**：
1. 业务管理 → 拒收记录
2. 查询：按企业/端口/号码筛选
3. 查看详情：选择记录 → 点击「详情」

#### 3.4.5 拒收关键词配置

**功能**：配置拒收识别关键词

**操作流程**：
1. 业务管理 → 拒收关键词配置
2. 修改关键词：在输入框填写（多个用逗号分隔）→ 点击「保存」

## 四、常用操作流程

### 4.1 新增合作方完整流程

1. **企业管理** → 新增企业 → 填写基础信息 + 信用代码 + 资质有效期 + 合作期限 + 联系人 + 经营范围 → 保存
2. **业务管理** → 合作期限台账 → 新增 → 选择企业 → 填写合作开始/结束日期 → 保存
3. **通道产品** → 产品管理 → 新增 → 填写号码信息 + 归属合作方 → 保存
4. **企业管理** → 用户管理 → 为企业创建用户账号

### 4.2 处理投诉完整流程

1. **业务管理** → 投诉处理 → 点击「新增」→ 登记投诉
2. 选择投诉 → 点击「处理」→ 更新状态为「处理中」→ 填写处理人/初步结果 → 保存
3. 处理完成后 → 点击「处理」→ 状态改为「已处理」→ 填写最终处理结果 → 保存
4. **业务管理** → 投诉统计 → 查看处理效率

### 4.3 配置上游通道

1. **通道产品** → 通道管理 → 新增
2. 选择协议（CMPP/SGIP/SMGP/HTTP）
3. 填写连接参数（IP/端口/账号/密码）
4. 测试连接 → 保存
5. **通道产品** → 产品管理 → 关联通道与号码

### 4.4 查看业务总览

1. **业务管理** → 业务看板 → 查看 KPI 和图表
2. 如需详细数据 → 点击快捷入口跳转对应台账
3. 如需统计分析 → 选择对应统计页

## 六、注意事项

### 6.1 页面风格
- 所有页面使用 **layui 原生风格**（白色背景、layui-card 布局、layui-btn 按钮）
- 统计页面和看板保留 ECharts 图表，使用系统主题色
- 风格与系统原有页面（代码字典、账户管理等）保持一致

### 6.2 数据安全
- 信用代码必须唯一，系统自动校验
- 资质过期的企业无法新增合作期限
- 拒收关键词修改后立即生效（影响上行短信处理）

### 6.3 操作规范
- 投诉状态只能正向流转（待处理 → 处理中 → 已处理）
- 合作期限"终止"是状态变更，不是删除（保留历史记录）
- 删除操作需谨慎，建议优先使用"终止/停用"

### 6.4 系统维护

**一键部署脚本**：
```bash
# 本地执行（自动完成 rsync + 编译 + 部署）
bash deploy.sh
```

**手动部署**：
```bash
# 服务器执行
export JAVA_HOME=/tools/jdk8
cd /tools/sms/sms_platform
mvn clean package -DskipTests -q
bash /tools/deploy-sms.sh
```

**数据库备份**：定期备份 sms_business 和 sms_send 库

**代码仓库**：https://github.com/wangganglab/sms-gateway

### 6.5 常见问题

| 问题 | 解决方案 |
|------|----------|
| 登录失败 | 检查账号密码/账号状态/登录失败次数 |
| 菜单不显示 | 检查角色权限分配（权限管理） |
| 页面 404 | 检查 JSP 文件是否部署到 Tomcat |
| 数据不显示 | 检查筛选条件/数据库连接/服务状态 |
| 投诉无法新增 | 检查必填字段（企业/号码/内容） |
| 菜单重复 | 数据库已修复，刷新浏览器即可 |
