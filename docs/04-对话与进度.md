### 阶段 8：信通畅邮部署成功（2026-06-29）

按 docs/03 部署，实际结构与原假设多处出入，逐一踩坑修复，最终部署成功。

**部署结果**：
- 3 平台公网可访问：admin(8889) / enterprise(8890) / netway(8888)
- 运营平台 http://36.139.116.238:8889/public/admin/login.jsp（admin/123456）
- DB：sms_business(41表) / sms_send(11表)，专用用户 `sms/smsGateway2026`
- 3 Tomcat 实例 0 错误，Druid 双数据源连接成功

**关键路径**：clone → 建库 → sed 修 SQL(ROW_FORMAT) → 导入 → 改 datasource.properties → mvn package → 3 WAR 部署到 3 Tomcat ROOT → 启动

**踩坑与解决**：
1. 配置文件叫 `datasource.properties`（非 README 写的 database.properties，README 自己写错）
2. 工程嵌套结构：`sms-system/sms-admin/sms-admin-web`（非平级三工程）
3. sms-quick 1.0.15 私有依赖（Maven 中央仓库无）→ gitee clone quick-sms + mvn install，主 pom 版本 1.0.9→1.0.15
4. **Java 17 不兼容**（javax.xml.bind/JAX-WS/sun.misc.BASE64Encoder 移除）→ 切 Temurin Java 8（Debian 12 无 openjdk-8，清华镜像下 binary 到 /tools/jdk8）
5. Debian 12 无 tomcat9（只有 tomcat10，jakarta 命名空间不兼容 javax）→ 清华镜像下 Tomcat 9.0.119 binary 到 /tools/tomcat9
6. MariaDB root 仅 socket 免密，JDBC 走 TCP 3306 被拒(ERROR 1698) → 建专用 `sms` 用户
7. **business.sql 首次导入只建 24/41 表**（旧库残留致 admin_limit id=1 duplicate，mysql 后续中止）→ DROP + `mysql --force` 干净重导，41 表全建
8. Tomcat 多实例：3 份 tomcat9 副本(/tools/tms-netway|admin|enterprise)，各改 server.xml 三端口 + setenv.sh 指向 Java 8
