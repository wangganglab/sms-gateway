### 清理 playSMS（如需）
```bash
# 停守护
crontab -l | grep -v playsmsd | crontab -
pkill -f playsmsd
# 删文件
rm -rf /opt/playsms /var/www/playsms /opt/playsms-data
# 删 apache 配置
a2dissite playsms.conf && systemctl reload apache2
# 删库
mysql -u root -e "DROP DATABASE playsms; DROP USER 'playsms'@'localhost';"
```

## 信通畅邮部署计划（待执行）

参考 README：环境需 JDK1.8+ / MySQL8.0+ / Maven3.0+ / Tomcat9.0+。当前 Java17 满足，MariaDB10.11 大部分兼容。

### 步骤

```bash
# 1. clone 到 /tools/sms
cd /tools/sms
git clone https://github.com/liuyanning/sms_platform.git

# 2. 建库（utf8mb4）
mysql -u root -e "CREATE DATABASE sms_business DEFAULT CHARSET utf8mb4;
                  CREATE DATABASE sms_send DEFAULT CHARSET utf8mb4;"

# 3. ⚠️ 修复 SQL 兼容（关键！上次卡这）
cd sms_platform
# 把 ROW_FORMAT=COMPACT 改成 DYNAMIC，解决 MariaDB 行大小超限
sed -i 's/ROW_FORMAT=COMPACT/ROW_FORMAT=DYNAMIC/g' docs/sql/business.sql
sed -i 's/ROW_FORMAT=COMPACT/ROW_FORMAT=DYNAMIC/g' docs/sql/send.sql
# 导入
mysql -u root sms_business < docs/sql/business.sql
mysql -u root sms_send < docs/sql/send.sql

# 4. 改 database.properties（3 个工程都改）
#    sms-admin / sms-enterprise / sms-netway-sender 下的 src/main/resources/database.properties
#    改 DB 地址/账号/密码（root 无密码）

# 5. Maven 构建
mvn clean package -DskipTests

# 6. 装 Tomcat 9，部署 3 个 WAR 到各自 ROOT
#    sms-netway-sender（发送基础，必须先启动）
#    sms-admin（运营管理平台）
#    sms-enterprise（客户端平台）

# 7. 访问运营平台
#    http://ip:port/public/admin/login.jsp
```

### 上次卡点复盘

| 问题 | 原因 | 解决 |
|------|------|------|
| business.sql 导入报 `Row size too large (> 8126)` | 建表 `ROW_FORMAT=COMPACT`，VARCHAR(512/2048) 多列导致行超 8126 字节 | sed 改 `ROW_FORMAT=DYNAMIC`（InnoDB 动态行格式把溢出列存页外） |
| `SET innodb_strict_mode=0` 无效 | 只放宽校验，没改行格式，表仍建在 COMPACT | 必须改 SQL 本身的 ROW_FORMAT |

## 实际部署结果（2026-06-29，✅ 成功）

部署完成。与原计划的差异和踩坑详见 `docs/04-对话与进度.md` 阶段 8。关键差异：

- **Java**：实际用 Temurin **Java 8**（`/tools/jdk8`），非服务器默认 17。Java 17 与项目不兼容，必须 Java 8。
- **Tomcat**：9.0.119 **binary**（`/tools/tomcat9`），Debian 12 apt 无 tomcat9（只有 tomcat10，jakarta 命名空间不兼容 javax）。
- **配置文件**：`datasource.properties`（**非** README 写的 database.properties）。
- **私有依赖**：sms-quick 1.0.15（`com.drondea.sms`），Maven 中央仓库无，从 gitee.com/zhuang-dian/quick-sms clone + `mvn install`；主 pom 版本 1.0.9→1.0.15。
- **DB**：用专用 `sms/smsGateway2026` 用户（root 仅 socket 免密，JDBC 走 TCP 3306 被拒 ERROR 1698）。
- **SQL 导入**：business.sql 须 `mysql --force` 导入到**干净库**(DROP 重建)，否则旧库残留致 admin_limit duplicate，mysql 中止后只建 24/41 表。
- **3 Tomcat 实例**：`/tools/tms-{netway,admin,enterprise}`，端口 8888/8889/8890，各 setenv.sh 指向 Java 8。
- **运维脚本**：`/tools/deploy-sms.sh`（部署）、`/tools/restart-sms.sh`（重启）、日志 `/tools/tms-<name>/logs/catalina.out`。

### 启停命令
```bash
# 启动（netway 必须最先）
for t in netway admin enterprise; do /tools/tms-$t/bin/startup.sh; done
# 停止

```
